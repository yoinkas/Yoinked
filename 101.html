command line exercises

Receiving files and traveling through directories

This lab gives you a fake terminal with dummy directories, flags, and simple questions. Work through it in order and use the ? button in the top-right corner when you need a hint.

starter-terminal

student@yoinkas:/home/student$

Walkthrough

  1. Use pwd to see where you are.
  2. Use ls to see what is available.
  3. Move into the missions directory.
  4. Read welcome.txt.
  5. Find the first flag in first-flag.txt.
  6. Question: what command takes you back one directory? Use it now.
  7. Move into Downloads.
  8. Open incoming.flag and collect the second flag.
  9. Use ls -la to reveal hidden files.
  10. Read .secret to collect the final flag.

quizzes

Beginner-friendly questions

Use these to check whether the basics are starting to stick. Try answering them before revealing the answers.

Quiz 1

Question: What command shows you your current working directory?

Answer: pwd

Quiz 2

Question: What does FTP do wrong from a security perspective?

Answer: It sends credentials and traffic in plaintext.

Quiz 3

Question: What command is commonly used to list files in a directory on Linux?

Answer: ls

Quiz 4

Question: Why is a password like password123 weak?

Answer: It is common, predictable, and easy to guess with a dictionary attack.

matching game

Match the term to the definition

Read the terms on the left and match them to the correct meaning. This is a simple memory drill for beginner cyber vocabulary.

Directory

A folder used to organize files.

FTP

A file transfer protocol that is insecure because it does not encrypt traffic by default.

Packet Capture

A recorded set of network traffic that can be reviewed in tools like Wireshark.

Brute Force

A method of trying many passwords or values until one works.

Web Shell

A malicious script uploaded to a web server to execute commands remotely.

Privilege Escalation

The act of gaining higher-level permissions after initial access.
